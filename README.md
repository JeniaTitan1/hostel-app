# 🏛️ Система розселення студентів у гуртожитках МНАУ

Система розселення студентів та управління гуртожитками для **Миколаївського національного аграрного університету (МНАУ)**. Проєкт забезпечує повний цикл управління корпусами, кімнатами, заявками на проживання, ордерами на поселення, ремонтними роботами, академічними довідниками та обліковими записами з підтримкою ролей комендантів та адміністраторів.

---

## 🌟 Ключовий функціонал системи

### 1. 👥 Багаторівнева система ролей та прав доступу
- **Адміністратор (`admin`)**: Повний доступ до всіх корпусів, кімнат, системних налаштувань, академічних довідників, модерації змін email, журналу аудиту, керування комендантами та імперсонації користувачів.
- **Комендант (`commandant`)**: Окремий робочий простір для керування конкретним корпусом (`building_id`). Має доступ до 2D-карти свого корпусу, жильців, заявок на розселення/переселення та ремонтних робіт у межах свого корпусу.
- **Студент (`user`)**: Вибір та бронювання вільного ліжко-місця, генерація та завантаження електронного **Ордера на поселення** з QR-кодом, подання заявок на ремонт, розгляд пропозицій переселення, налаштування профілю та сповіщення.

### 2. 🗺️ 2D-Візуалізатор поверхів (Floor Map Visualizer)
- Інтерактивна схема гуртожитків з групуванням по поверхах та кольоровими індикаторами заповненості й станів.
- **Гнучке управління станами кімнат**:
  - **Статус ремонту (`status`)**: Переведення кімнати в статус `closed` (Капітальний ремонт) з вказанням причини та очікуваного терміну завершення.
  - **Закриття набору (`intake_closed`)**: Заборона нового бронювання кімнати без виселення вже проживаючих студентів.
  - **Приховання з фронтенду (`hide_from_frontend`)**: Кімната приховується від студентів (видима лише адміністрації).
  - **Динамічна місткість (`max_capacity`)**: Зміна кількості ліжко-місць у кімнаті.
- **Статевий контроль (Gender policy)**: Автоматична перевірка та розділення за статтю для запобігання змішаному розселенню.

### 3. 📄 Ордери на поселення та QR-код перевірка (Order Verification)
- Автоматична генерація електронного ордера на поселення після схвалення заявки.
- Кожен ордер містить унікальний номер, герб/символіку, дані студента, корпус, номер кімнати та дату поселення.
- **Вбудований QR-код**: Веде на публічну сторінку перевірки автентичності ордера (`/verify-order/{orderNumber}`), що дозволяє комендантам або охороні швидко перевірити чийсність ордера зі смартфона.

### 4. 🔄 Система пропозицій переселення (Reallocation Workflow)
- Адміністратор або комендант може запропонувати студенту переселення в іншу кімнату.
- Студент отримує персональне сповіщення та інтерактивний банер у кабінеті з можливістю підтвердити або відхилити переселення.

### 5. 📧 Модерація зміни електронної пошти (Email Change Requests)
- Студенти подають запит на зміну адреси електронної пошти з вказуванням нової адреси.
- Запити відображаються у панелі адміністратора з можливістю схвалення або відхилення з вказанням причини.

### 6. 🏫 Академічні довідники та системні налаштування
- **Академічні опції**: Керування довідниками Спеціальностей (КН, ГРС, МЕН тощо), Курсів (1-6) та Академічних груп.
- **Глобальні налаштування**: Налаштування дозволених поштових доменів для реєстрації, керування прийомом заявок на рівні університету.

### 7. 👤 Керування комендантами та імперсонація
- Створення та генерація акаунтів комендантів із прив'язкою до конкретних корпусів.
- **Імперсонація (Impersonation)**: Можливість адміністратора тимчасово увійти в систему під виглядом будь-якого студента або коменданта для перевірки відображення чи вирішення питань користувача.

### 8. 🔔 Центр сповіщень (Notification System)
- Внутрішні сповіщення про зміну статусу заявки, ремонтні роботи, пропозиції переселення та результати модерації.
- Можливість поштучного або масового позначення сповіщень як прочитаних.

### 9. 🛠️ Журнал ремонтних заявок (Maintenance Tickets)
- Студенти з активним поселенням можуть подавати заявки на ремонт з описом проблеми.
- Адміністратори та коменданти бачать заявки свого корпусу та позначають їх як виконані (`resolved`).

### 10. 📊 Журнал дій (Audit Log)
- Автоматична фіксація всіх дій в системі (заселення, переселення, виселення, зміна параметрів кімнат, створення корпусів).
- Функція очищення журналу аудиту для адміністратора.

### 11. 🎲 Генератор користувачів & Примусова активація
- Пакетна генерація тимчасових акаунтів (типу `studentXXXX@mnau.edu.ua`) з експортом у PDF чи копіюванням у буфер.
- Middleware примусового блокування тимчасових акаунтів на сторінці профілю до заповнення справжніх даних та зміни пароля.

## 🛠️ Технологічний стек

- **Backend**: PHP 8.3+, Laravel 11 / 13, Inertia.js (Laravel Adapter)
- **Frontend**: React 18, Inertia.js React, Tailwind CSS v4, Headless UI, Lucide Icons
- **PDF & QR**: jsPDF, html2canvas, SVG / Dynamic QR generation
- **База даних**: MySQL / SQLite
- **Контейнеризація**: Docker, Laravel Sail

## 🐳 Інструкція з запуску в Docker (Laravel Sail)

### Крок 1. Підготовка оточення
1. Перейдіть до папки з проєктом:
   ```bash
   cd hostel-app
   ```
2. Створіть файл конфігурації `.env`:
   ```bash
   cp .env.example .env
   ```

### Крок 2. Встановлення залежностей Composer (якщо немає локального PHP)
```bash
docker run --rm \
    -u "$(id -u):$(id -g)" \
    -v "$(pwd):/var/www/html" \
    -w /var/www/html \
    laravelsail/php83-composer:latest \
    composer install --ignore-platform-reqs
```

### Крок 3. Запуск контейнерів
```bash
./vendor/bin/sail up -d
```

### Крок 4. Генерація ключа додатку
```bash
./vendor/bin/sail artisan key:generate
```

### Крок 5. Міграції та наповнення бази даних
```bash
./vendor/bin/sail artisan migrate:fresh --seed
```

### Крок 6. Встановлення npm-залежностей та запуск фронтенду
```bash
./vendor/bin/sail npm install
./vendor/bin/sail npm run dev
```

### Крок 7. Перегляд проєкту
Відкрийте у браузері: 👉 **[http://localhost](http://localhost)**

## 💻 Локальний запуск (без Docker)

Якщо на вашому комп'ютері встановлені PHP 8.3+, Composer та Node.js:

1. **Встановлення залежностей Composer та NPM**:
   ```bash
   composer install
   npm install
   ```
2. **Конфігурація та бази даних**:
   ```bash
   cp .env.example .env
   php artisan key:generate
   php artisan migrate:fresh --seed
   ```
3. **Запуск серверів розробки**:
   ```bash
   npm run dev
   # у другому терміналі:
   php artisan serve
   ```
